# Архитектурное описание системы управления задачами (Todo-лист)

## Назначение системы
Система позволяет пользователям создавать проекты, добавлять в них задачи, управлять статусами задач (выполнена/не выполнена), устанавливать сроки, приоритеты и просматривать списки задач по проектам и по статусам. Реализована с применением принципов ООП: инкапсуляция данных, разделение ответственности между классами, взаимодействие объектов через чётко определённые интерфейсы.

## Основные классы

### 1. `Task` (Задача)
**Назначение:** описывает отдельную задачу.

**Атрибуты:**
- `id` — уникальный идентификатор задачи (int).
- `title` — название задачи (str).
- `description` — подробное описание (str, может быть пустым).
- `project_id` — идентификатор проекта, к которому относится задача (int или None).
- `is_done` — флаг завершённости (bool, по умолчанию False).
- `due_date` — дата и время дедлайна (datetime или None).
- `priority` — приоритет задачи (str: "low", "medium", "high" или int: 1–3).
- `created_at` — дата создания задачи (datetime, устанавливается автоматически).

**Методы:**
- `mark_as_done()` — помечает задачу как выполненную.
- `mark_as_undone()` — снимает отметку о выполнении.
- `update_description(new_text: str)` — обновляет описание.
- `set_due_date(date: datetime)` — устанавливает или обновляет дедлайн.
- `set_priority(priority: str | int)` — устанавливает приоритет.
- `to_dict()` — возвращает представление задачи в виде словаря (удобно для сериализации, например, в JSON).

---

### 2. `Project` (Проект)
**Назначение:** группирует задачи по смысловым блокам.

**Атрибуты:**
- `id` — уникальный идентификатор проекта (int).
- `name` — название проекта (str).
- `owner_id` — идентификатор владельца проекта (int).
- `tasks` — список объектов `Task`, относящихся к проекту (list[Task]).
- `created_at` — дата создания проекта (datetime).

**Методы:**
- `add_task(task: Task)` — добавляет задачу в проект и связывает её по `project_id`.
- `remove_task(task_id: int)` — удаляет задачу из проекта (не обязательно удаляет объект, а убирает из списка и очищает `project_id`).
- `get_tasks(filter_done: bool | None = None, filter_priority: str | None = None)` — возвращает список задач с фильтрацией по статусу и приоритету.
- `get_overdue_tasks()` — возвращает список просроченных задач.
- `rename(new_name: str)` — изменяет название проекта.
- `to_dict()` — сериализует проект (без глубокой сериализации задач или с опцией включить их).

---

### 3. `User` (Пользователь)
**Назначение:** представляет пользователя системы, владеет проектами и задачами.

**Атрибуты:**
- `id` — уникальный идентификатор пользователя (int).
- `username` — имя пользователя (str).
- `email` — электронная почта (str).
- `projects` — список проектов, которыми владеет пользователь (list[Project]).

**Методы:**
- `create_project(name: str) -> Project` — создаёт новый проект и добавляет его в список `projects`.
- `get_projects()` — возвращает все проекты пользователя.
- `get_all_tasks()` — собирает все задачи пользователя из всех его проектов (удобно для общего списка «Мои задачи»).
- `to_dict()` — представление пользователя в виде словаря.




## Взаимодействие объектов

- **Пользователь создаёт проект:** вызывается `user.create_project("Новый проект")`. Создаётся объект `Project`, добавляется в `user.projects`.
- **Добавление задачи в проект:** создаётся объект `Task`, затем вызывается `project.add_task(task)`. Внутри метода `add_task` устанавливается `task.project_id = project.id`, и задача добавляется в список `project.tasks`.
- **Изменение статуса задачи:** у объекта `Task` вызывается `task.mark_as_done()`, что меняет `is_done = True`. Это изменение сразу видно и в проекте (`project.get_tasks()`), и при глобальном поиске.
- **Фильтрация и отчёты:** `Project.get_tasks(...)` фильтрует задачи внутри проекта. `TaskManager.get_all_overdue()` собирает задачи со всех проектов для отчёта «Просроченные задачи».
- **Удаление задачи:** `project.remove_task(task_id)` убирает задачу из списка проекта и обнуляет `project_id`, но сам объект может оставаться в глобальном хранилище (если используется `TaskManager`) либо удаляться полностью — в зависимости от выбранной политики хранения.
