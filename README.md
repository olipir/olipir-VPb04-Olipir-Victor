# Система управления задачами (Todo-лист)

Проект реализует систему управления задачами на Python с использованием принципов ООП. Пользователи создают проекты, добавляют в них задачи, управляют статусами, сроками и приоритетами.

## Структура проекта

| Файл | Назначение |
|------|------------|
| `architecture,md` | Архитектурное описание системы |
| `task_management.py` | Реализация классов и демонстрация работы |
| `manifest.md` | Отчёт о выполнении задания |

---

## `architecture,md`

Архитектурный документ, описывающий проектируемую систему до написания кода.

**Содержит:**
- назначение системы;
- описание классов `Task`, `Project`, `User` с атрибутами и методами;
- схему взаимодействия объектов (создание проектов, добавление задач, фильтрация, отчёты через `TaskManager`).

Этот файл служит техническим заданием для генерации кода и фиксирует структуру системы на этапе проектирования.

---

## `task_management.py`

Основной файл с реализацией системы на Python.

**Классы:**

| Класс | Описание |
|-------|----------|
| `Task` | Отдельная задача: название, описание, дедлайн, приоритет, статус выполнения |
| `Project` | Группа задач: добавление, удаление, фильтрация, поиск просроченных |
| `User` | Пользователь: создание проектов, сбор всех задач |
| `TaskManager` | Глобальное хранилище задач для поиска и отчётов |

**Запуск демонстрации:**

```bash
python task_management.py
```

При запуске выполняется пример: создаётся пользователь, проекты и задачи, демонстрируются основные методы системы.

---

## `manifest.md`

Краткий отчёт о выполнении задания.

**Содержит:**
- промпт, отправленный в Cursor для генерации кода;
- оценку результата (соответствие ТЗ, отсутствие ошибок);
- вывод о пользе предварительного описания архитектуры в `architecture,md`.

---

## Порядок работы с проектом

1. Изучить архитектуру в `architecture,md`.
2. Запустить `task_management.py` и посмотреть демонстрацию.
3. Ознакомиться с отчётом в `manifest.md`.
